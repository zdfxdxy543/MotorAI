Detailed Description

A collection of functions to determine ADC bias values from circuit parameters.

The bias value is typically represented in a per-unit (p.u.) system, where the value is normalized with respect to the ADC's reference voltage (Vref). These models help in calculating the expected digital offset for various analog sensor front-end circuits. The calculated bias is essential for correctly interpreting the ADC output.

GMP_STATIC_INLINE parameter_gt ctl_bias_calc_via_res_divider ( parameter_gt R1,
parameter_gt R2 )

Calculates the bias generated by a resistive voltage divider.

This model is for circuits where the bias is set by a voltage divider connected between Vref and GND. The formula is: \(\text{bias}_{\text{p.u.}} = \frac{R_2}{R_1 + R_2} \)

Parameters
R1The resistance of the resistor connected between Vref and the output tap.
R2The resistance of the resistor connected between the output tap and GND.
Returns
The calculated bias in per-unit.

GMP_STATIC_INLINE parameter_gt ctl_bias_calc_via_Vref_Vbias ( parameter_gt Vref,
parameter_gt Vbias )

Calculates the ADC bias in per-unit based on the reference and bias voltages.

This is the most direct way to calculate bias if the absolute voltages are known. The formula is: \(\text{bias}_{\text{p.u.}} = \frac{V_{\text{bias}}}{V_{\text{ref}}} \)

Parameters
VrefThe ADC reference voltage.
VbiasThe bias voltage applied to the sensor or ADC input.
Returns
The calculated bias in per-unit.
