# DPE (软件定义电力电子) 配置工具项目开发书

## 1. 项目概述

**项目名称：** SDPE (Software-Defined Power Electronics) 配置与代码生成工具

**核心目标：** 构建一个高度解耦的电力电子控制系统“数据底座”。通过可视化界面管理硬件拓扑与物理参数，自动生成针对具体 MCU/DSP 的底层 C 语言初始化代码与回调脚手架，使控制算法开发彻底脱离底层硬件的强绑定。

**设计哲学：** * **物理与软件隔离：** 用户输入物理量（如电阻、电压），系统自动转化为软件变量与宏。

- **不完全接管：** 专注数据定义与数据流向，保留 `USER CODE` 保护区，将 DMA 调度与核心控制逻辑（如 FOC、SVPWM 算法）的自由度交还给开发者。

## 2. 系统架构设计 (核心数据模型)

系统采用高度解耦的“五层 JSON 描述架构”，从底层数据类型一路贯穿至最终的板级产品。

| **层级**          | **文件后缀**        | **核心职责**                                                 | **目标用户**       |
| ----------------- | ------------------- | ------------------------------------------------------------ | ------------------ |
| **1. 类型字典**   | `sdpe_typedef.json` | 定义全局数据基石（如 `ctrl_gt`, `fast_gt`）及其在 C 语言中的字面量包装规则。 | 底层架构师         |
| **2. 元件元数据** | `*.meta.json`       | 描述 C 语言库函数（如 `adc_channel_t`），暴露可配置的参数、输入输出端口及初始化模板（如电阻分压 vs 运放采样）。 | 库开发者           |
| **3. 装置蓝图**   | `*.bp.json`         | 定义硬件拓扑（如“三相逆变器”必须包含三相全桥和电流传感器），约束基础参数，导出系统特征宏。 | 硬件系统工程师     |
| **4. 装置实例**   | `*.inst.json`       | 蓝图的具体物化。由用户填入具体的物理参数（如分压电阻实际阻值），实现模板延迟绑定。 | 应用算法工程师     |
| **5. 产品集成**   | `*.prod.json`       | 系统级组装。定义外设宏（如 TIM1, ADC1），选择回调范式，将纯软件输入输出端口与物理寄存器或 DMA 内存地址硬绑定。 | BSP/固件集成工程师 |

## 3. 技术栈选型

- **宿主环境与框架：** Electron。提供纯正的桌面级文件系统访问权限，能够无缝读写本地 C 工程目录。
- **前端 UI 框架：** Vue.js (或 React)。利用其强大的数据双向绑定机制，轻松管理深层嵌套的 JSON 树与动态表单。
- **代码编辑器组件：** Monaco Editor (VS Code 核心)。为库管理器和实时代码预览提供专业的 C 语言语法高亮与代码提示。
- **代码生成引擎：** 基于 Node.js 的模板引擎（如 EJS 或 Jinja2 的 JS 移植版），负责将解析后的数据填充至 `.c` 和 `.h` 模板中。

## 4. 用户界面 (UI) 模块规划

软件界面分为四个主要工作区，通过左侧全局导航栏切换：

- **库管理器 (Library Manager)：** 用于编辑 `typedef` 和 `meta` 文件。提供模板代码的高亮编辑器与占位符变量检测器。
- **蓝图架构师 (Blueprint Architect)：** 列表与属性面板的组合视图。用于将组件拖拽组合成逻辑板卡，定义全局宏依赖。
- **实例配置器 (Instance Configurator)：** 极其简洁的向导式表单。只显示当前蓝图暴露的物理参数输入框，并在后台强制执行“值快照”以防止底层库更新导致的项目崩溃。
- **产品路由矩阵 (Routing Matrix)：** 交叉分配矩阵视图。将 `inst` 中悬空的逻辑端口（如 `raw_a`）与目标芯片的寄存器/DMA 变量进行可视化连线绑定。
- **全局悬浮功能：** 实时代码预览窗。在用户修改任何参数时，实时展示即将生成的 C 语言代码片段。

## 5. 代码生成规范

- **内存策略：** 所有结构体（如 `SDPE_t`）均采用全局静态分配，严禁在生成的代码中使用动态内存。
- **输出产物：** * `sdpe_config.h` / `.c`: 包含所有导出的宏定义、复合结构体定义及静态实例化。
  - `sdpe_main.c`: 包含完整的 `SDPE_Init()` 初始化调用链，以及带有 `/* USER CODE BEGIN/END */` 保护区的输入、输出、主循环回调脚手架。
- **宏防冲突：** 所有的实例参数宏与外设宏，强制加上产品或实例名称作为前缀（如 `PROD_M1_ADC_MOD`）。

## 6. 阶段开发计划 (Roadmap)

### 阶段一：核心解析器与代码生成 (Core CLI & Parser)

- 脱离 UI，使用 Node.js 编写纯脚本。
- 手工构造一套完整的五层 JSON 测试样例（ADC + Inverter）。
- 完成 JSON 解析、上下文变量池构建、模板渲染，成功生成可编译的 `sdpe_main.c` 与头文件。

### 阶段二：UI 基础框架与数据绑定 (Frontend Scaffold)

- 搭建 Electron + Vue/React 工程。
- 实现本地文件系统的目录读取与 JSON 文件的存取逻辑。
- 引入 Monaco Editor 跑通代码高亮组件。

### 阶段三：四大工作区业务逻辑开发 (Workspaces)

- 逐步实现库管理器、蓝图架构师、实例配置器、产品路由矩阵的 UI 交互。
- 打通 UI 操作与后台生成脚本的链路，实现“修改参数 -> 实时预览更新”。

### 阶段四：工程化与测试 (Engineering & Validation)

- 引入 JSON Schema (如 Ajv 库) 实现严格的数据合法性校验。
- 将生成的代码放入实际的 STM32 或 DSP 工程中进行编译与硬件在环测试。
